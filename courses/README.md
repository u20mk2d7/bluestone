## 4. Architectural Target: The HFT Limit Order Book

A quantitative infrastructure engineer must understand how financial logic maps directly into software layout. When designing an institutional-grade Limit Order Book (LOB), traditional software engineering patterns fail due to memory fragmentation and pointer indirection.

```
                  ┌─────────────────────────────────┐
                  │    Price-Time Priority Engine   │
                  └────────────────┬────────────────┘
                                   │
         ┌─────────────────────────┴─────────────────────────┐
         ▼                                                   ▼
┌───────────────────┐                               ┌───────────────────┐
│     Bids Tree     │                               │     Asks Tree     │
│ (Max-Heap/Array)  │                               │ (Min-Heap/Array)  │
└────────┬──────────┘                               └────────┬──────────┘
         │                                                   │
         ├─► Price Level 1 ($100.50)                         ├─► Price Level 1 ($100.52)
         │   └── Doubly-Linked List                          │   └── Doubly-Linked List
         │       ├── Order A (100 Shares)                    │       ├── Order C (500 Shares)
         │       └── Order B (250 Shares)                    │       └── Order D (150 Shares)
         │                                                   │
         └─► Price Level 2 ($100.49)                         └─► Price Level 2 ($100.55)
```

### Strategic Layout Rules:
* **The Pointer Antipattern:** Avoid standard balanced trees like `std::map` or `std::set` in the core matching loop. Node-based structures generate non-contiguous allocations, inducing catastrophic CPU cache misses during order book traversal.
* **Contiguous Arrays & Flat Structures:** Utilize arrays or dense vectors representing explicit price levels, backed by custom block memory allocators to recycle order objects instantly.
* **Data Fields:** Prioritize strict FIFO (First-In, First-Out) tracking at each explicit price tier using lightweight, inline structures.
